다음 순서로 배포를 진행해줘:

1. **변경 파일 확인**: `git status --short` 실행 → 변경/신규 파일 목록 확인
2. **테스트 여부 판단**:
   - 변경 파일이 모두 `.md` 파일인 경우 → 테스트 생략, 바로 3번으로
   - `.md` 외 파일이 하나라도 있는 경우 → 변경/신규 파일 실행 (파일 선택)
     - **실패 시**: 배포 중단, 오류 내용 보고
     - **통과 시**: 3번으로
3. **파일별 개별 커밋**: 변경/신규 파일 각각에 대해 아래를 반복:
   - `git add <파일명>` (파일 하나씩)
   - 해당 파일의 변경 내용을 반영한 영어 커밋 메시지 작성 후 `git commit`
4. **배포**: 모든 파일 커밋 완료 후 `git push`
