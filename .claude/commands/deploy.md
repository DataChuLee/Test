다음 순서로 배포를 진행해줘:

1. **변경 파일 확인**: `git status --short` 실행 → 변경/신규 파일 목록 확인
2. **테스트 여부 판단**:
   - 변경 파일이 모두 `.md` 파일인 경우 → 테스트 생략, 바로 3번으로
   - `.md` 외 파일이 하나라도 있는 경우 → 변경/신규 파일 실행
     - **실패 시**: 배포 중단, 오류 내용 보고
     - **통과 시**: 3번으로
3. **커밋 메시지 생성**: 변경 파일 목록을 보고 내용에 맞게 한국어로 간결하게 작성
4. **스테이징**:  변경된 파일들 명시적으로 `git add`
5. **배포**: `git commit` → `git push`
